Большую сложность написания хедера движка под С++ составляло огромнейшие отличия между разными компиляторами... Самый же капризный оказался MS Visual C++ версия 6 и ниже(возможно с 2005 и т.д. проблем не было бы). Ну да ладно, главное что удалось реализовать хедеры.

Итак, файлы:

"OMEGA_code.cpp" - должен быть подключен только к главному(!) файлу проекта посредством директивы #include, или средствами среды программирования(например в Borland C++Builder есть "Project->Add to Project", а в MS Visual C++ "Project->Add to project->Files")

"OMEGA_functions.h" - файл который должен подключаться к любому другому модулю который использует функции движка. Также надо подключить и к главному модулю если захочется использовать функции с параметрами по-умолчанию.

"OMEGA_types" - вообще он по идеи должен подключаться ко всем модулям где используются типы движка, но возможно это не потребуется из-за того что подключив OMEGA_code.cpp к главному модулю, эта потребность исчезнет.

Теперь про файлы ООП-оболочки:

"OMEGA_classes.cpp" - должен быть использован также как и "OMEGA_code.cpp"

"OMEGA_classes.h" - полная аналогия OMEGA_types.h, и поэтому его подключение тоже может и не потребоваться когда модуль будет использовать описание классов

************************************************************
Предупреждения пользоватеям Borland C++ Builder:
Если вы используете VCL-режим создания проектов, тогда вам придется сменить названия типа TTimer на какой-то свой. Это проделать стоит командой Search->Replace с параметром All.

Постоянно задаваемый вопрос программистами использующих С++:
"Где lib-файл?"

Ответ:
"Библиотека пишется на Delphi и поэтому этот файл не генерируется данной средой."

Может сразу появится встречный вопрос:
"Как же импортируются функции из библиотеки?"

Ответ:
"Хедеры реализуют динамическое подключение библиотеки, где функции представляют собой простые типизированные указатели, которые получают адреса на настоящие функции после вызова LoadOMEGA(). А в хедере OMEGA_code.cpp реализуются inline-функции чтоб было видно синтаксис при вызове функций и можно было задать параметры по умолчанию(вот этот момент пришлось реализовывать именно для MS Visual C++, когда Borland C++ Builder работает и с простыми указателями)"
