Этот плагин демонстриует как использвать функцию движка для создания некоторых деталей взаимодействия разрабатываемого плагина. Ничего сложного в этом нет, вся проблема заключается в той идеи которую вы хотите реализовать.

Данный плагин реализует простейшую "технологию" 2D-освещения. Суть такова: на экране есть определенное количество квадратов с одинаковым размером. Источник света(пока только круг) определяет какие вершины квадратов входят в него и задает им определенную альфу. Альфа тех вершин что находятся ближе к краю окружности источника света устанавливается так чтоб было видно какой-то плавный переход. Вот и вся логика... Сам же плагин реализует менеджер для управления этими источниками света и все просчеты берет на себя. Каждый реализует подобный код по своему, я же решил его оптимизировать, незначително... Оптимизация заключается в исключение кавадратов которые явно не попадут в окружность источника света. Плагину нехватает алгоритма который бы рисовал не все квадраты по отдельности а обьединял(хотяб темные) в один квад, т.е. допустим у нас есть только один источник света по центру экрана. Тогда в этом случаи надо нарисовать по отдельности только квадраты входящие в описывающий квадрат для этого источника, а для всего остального обойтись четырьмя большими прямогольниками с каждой из сторон.


Если что, то пишите на почтовик все свои пожелания, вопросы, разгромы, критику и т.д. Если есть желание использовать этот плагин как реальный инструмент, то я могу его дописать, используя более эффективную струткуру для хранения списка источников света.
